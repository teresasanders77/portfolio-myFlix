Documentation

This web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

The table below describes each endpoint of the myFlix API and how to access it.

ENDPOINT URL DESCRIPTION METHOD PARAMETERS REQUEST RESPONSE
/movies This endpoint returns a list of all movies to the user. GET None None JSON object: data about all of the movies.
movies/:title This endpoint retrieves data on a single movie (Description, genre, director, image URL) by movie title. GET None None JSON object: data about a single movie.
/genres/:name This endpoint retrieves data on a certain genre by name. GET None None JSON object: definition of a specific genre.
/directors/:name This endpoint provides information about a specific director (Bio, birth year, death year) by name. GET None None JSON object: data about a director.
/users This endpoint allows a new user to register (Username, password, email, date of birth). POST None JSON object: data about the user to add. JSON object: data about the user that was added, including an ID.
/users/:username This endpoint allows current users to update their info (Username, password, email, date of birth). PUT None None A text message indicating the user, username, password, email, and date of birth that was updated.
/users/:username/favorites/:title This endpoint allows a user to add a specific movie to their list of favorites. POST None None JSON object: Movie and ID that were successfully added to their list of favorites.
/users/:username/favorites/:title This endpoint allows a user to remove a movie from their list of favorites. DELETE None None A text message indicating whether the movie was successfully removed from their list of favorites.
/users/:username This endpoint allows a current user to de-register their information. DELETE None None A text message indicating whether the user was successfully deregistered.
